安装markdown preview

按下键Ctrl+Shift+p调出命令面板，找到Package Control: install Pakage这一项。搜索markdown preview，点击安装。

使用

Markdown Preview较常用的功能是preview in browser和Export HTML in Sublime Text，前者可以在浏览器看到预览效果，后者可将markdown保存为html文件。

preview in browser据称是实时的，但是实践上还是需要在st保存，然后浏览器刷新才能看到新的效果，好在markdown写得多的话也不需要每敲一行看一次效果。

语法高亮跟编辑器的主题有关，可以在Preferences ->Color Scheme找自己喜欢的主题。
关于目录生成，只要文章是按照markdown语法写作的。在需要生成目录的地方写[TOC]即可。

如果你这里没有看到目录而只是看到代码，说明不支持目录自动生成哈哈

打印成pdf

将markdown转换为pdf应该有很多种方法的。我没有再折腾，直接用谷歌浏览器虚拟打印功能生成。

(3-1)蓝牙操作

针对短距离设备通讯(10M),常用于连接耳机鼠标和移动通讯设备，功耗低，

与蓝牙相关的API，
BluetoothAdapter：该类对象代表了本地的蓝牙适配器
BluetoothDevice：代表了一个远程的Bluetooth设备

扫描已配对的蓝牙设备

  1. 在Androidmanifest.xml声明蓝牙权限

(2-3)用户定位

User Location：通过设备上的GPS或者利用基站，WIFI来获取用户的位置信息，经过处理可以得到用户的运行轨迹

关键API
LocationManager：管理Android的用户定位服务
LocationProviders：提供多种定位方案

定位方式分类：
1：GPS定位，使用GPS卫星进行定位，需要添加权限ACCESS_FINE_LOCATION
2，NETWORK定位：使用信号接收塔和WIFI接入点进行定位，需要添加权限ACCESS_FINE_LOCATION或者ACCESS_COARSE_LOCATION

  1. 获取最佳的Location Provider
  2. 追踪用户的位置

获取设备的providers列表
List providers=locationManager.getgetAllProviders();

获取设备适合的provider
Criteria：可以设置一系列的查询条件用于查找当前设备当中符合条件的Location Provider，可以设置满足应用程序需求的最佳的田间，然后使用getBestProvider得到最佳的Provider

追踪用户主要使用requestLocationUpdates中的两个参数lowTime，lowDistinct

谷歌Geocoding服务：是谷歌提供的一项服务，可以查找地址的经纬度，也可以查询经纬度所对应的地址，在模拟器上不能运行。找到了一个替代的解决方案，直接访问服务的方式获取数据

http://maps.googleapis.com/maps/api/geocode/json?address=SFO&sensor=false

address:地址
sensor：是否是传感器数据
latlng:经纬度
bounds：经纬度边界
region：国家代码
