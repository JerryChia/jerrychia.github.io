如何使用hexo搭建Blog，并部署到gitcafe

发表于 2015-01-10   |   分类于 Hexo   |     |   阅读次数

摘要

hexo是台湾一个本科生 tommy351 所做的(好惭愧…)，是一个基于Node.js的静态博客，通过特定的语法生成静态网页。而且hexo生成的静态网页可以直接放到GitHub Pages，GitCafe，SAE等平台上，部署简单方便。
网上已经有很多人写如何配置的Blog，本文参考各位大神的方法搭建博客，在此只提供本人的思路，和自己在搭建过程中遇到的问题。搭建过程主要还是参考网上的内容，但也有自己的思考。

我想实现在GitCafe上面部署的有自己二级域名，类似于XXX.gitcafe.com访问的静态博客，所以，我先介绍GitCafe。

1，如何在GitCafe上部署有自己二级域名的Blog

GitHub相信大家都听说过，它是目前世界上最流行的代码托管平台，它适合一些项目开发与团队合作，但是也可以作为一个空间。但是在国内貌似被墙了，有时访问不到。

GitCafe空间也推出GitCafe Pages服务，其配置，托管方式跟GitHub类似。在指定的条件下，我们可以得到一个XXX.gitcafe.com的域名，XXX为你的用户名。目前已经支持绑定域名，空间没有广告，使用Git管理代码。

部署方式参考：

GitCafe国内在线托管软件项目平台申请试用:附与Github简单比较

其中包括：
gitcafe的配置
SSH配置
成功连接GitCafe使用Git管理代码

注意：

  1. 你想要生成二级域名的项目名称只能与你的用户名相同，如果你觉得自己的用户名不太好，可以在GitCafe中进行设置，需要等一段时间才能生效。
  2. 操作完成一般需要十分钟左右才能访问你的二级域名页面
  3. 我在主分支master下没有部署成功，在分支gitcafe-pages中成功托管
  4. ssh文件可以放在别的文件夹

遇到的问题：

  1. 如果你执行hexo代码时，提示找不到主题，
    你查看是不是在hexo目录下theme文件夹下面缺少主题
    或者博客目录下的配置文件里 _config.yml 里面的主题引用错误，如果修改了之后还不能正常显示，一句话：卸载重装吧
  2. 中文显示乱码问题，使用Sublime打开，保存为uft-8格式
  3. git主题时，提示无法访问github，使用代理，代理配置参考
    http://html5gamedev.org/?p=399
  4. 如果出现【warning: LF will be replaced by CRLF】，这个是警告信息，如果你想屏蔽这些信息，在 hexo系列教程：（五）hexo博客的优化技巧续 中有解决的办法。
  5. 添加百度分析后，在你的站点的每个页面的左上角都会看到一个恶心的百度LOGO。你只能在『百度统计首页->网站列表->获取代码->系统管理设置->统计图标设置->显示图标』，把那个勾去掉。百度真是恶心，我准备还是用Google Analytics。
